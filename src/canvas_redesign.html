PROBLEMS

Through 5 interviews with students using Canvas (in fall of 2020), I boiled down the main problems with the current system to 3 main items:

  1. Disorganized and varying organization structures between classes.
  2. Too many options to click through in order to get to destination.
  3. Too much time spent on adjusting and figuring out each class structure.

SOLUTIONS

As a result, I decided to focus on 3 main ideas in order to create a learning management system that can aid students' work-flow while also allowing instructors to setup their classes intuitively and with similar organization structures:

  1. Pare down current heirarchy to only the bare essentials.
  2. Consistent organization between classes for both students and instructors.
  3. Intuitive and efficient use in order to maintain workflow without interruption.

SITE MAP 1

First site map done in Optimal Workshop.

After running card sorting with the same interviewees, the first hierarchical solution that I came up with split the content into three main categories:

  1. Agenda
  2. Class Information
  3. Profile

The 'agenda' category presents relevant class information in a meaningful way in order to aid students' workflow. The 'class information' category is for class-specific course work as well as grades, while the 'profile' category serves as an overall settings, preferences, and class history.

SITE MAP 2

Second site map made in Adobe Illustrator.

After tree testing 5 scenarios involving the same interviewees, I decided to further reduce the 'class information' options because there seemed to be confusion as to where certain items were located. Only 6 categories remained within 'class information':

  1. Class Home
  2. Assignments
  3. Modules
  4. Class Calendar
  5. Syllabus
  6. Grade

FINAL RESULT

Final result, created using HTML and SASS within 11ty.

Although the site is responsive and works on mobile, the individual class navigation requires a collapsing menu, and as a result, I opted to leave the nav bar as is because I wanted to focus on the site hierarchy and overall design. Going forward, I plan to implement a collapsing menu for both the whole site navigation as well as for the individual class sites.
